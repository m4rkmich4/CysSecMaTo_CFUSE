Getting Started

Einführung

C-FUSE ist ein lokal ausführbares Analyse- und Mapping-System für Cybersecurity-Standards. Es kombiniert eine Neo4j-Graphdatenbank, KI-gestützte Ähnlichkeitsbewertungen und ein modulares Qt-Frontend zur Visualisierung und Steuerung der Analyseprozesse.

Dieses Kapitel zeigt, wie das Projekt installiert, konfiguriert und gestartet wird.

Repository herunterladen

Das Projekt ist öffentlich verfügbar unter:

https://github.com/m4rkmich4/CysSecMaTo_CFUSE.git

Klonen Sie das Repository in ein lokales Verzeichnis Ihrer Wahl:

git clone https://github.com/m4rkmich4/CysSecMaTo_CFUSE.git
cd CysSecMaTo_CFUSE

Systemvoraussetzungen

  • Python: Version 3.11 oder neuer

  • Abhängigkeiten: siehe requirements.txt

  • Neo4j: Version 5.x (lokaler Docker-Container empfohlen)

  • Betriebssystem: macOS, Linux oder Windows

  • Optional: Lokales LLM über Ollama, wenn KI-Funktionen genutzt werden

Installation

  1. Projektverzeichnis betreten

cd /Pfad/zum/Projekt/CysSecMaTo_CFUSE

  1. Virtuelle Umgebung erstellen

macOS / Linux:

python3 -m venv .venv
source .venv/bin/activate

Windows (CMD):

python -m venv .venv
.\.venv\Scripts\activate

  1. Abhängigkeiten installieren

pip install -r requirements.txt

  1. Neo4j konfigurieren

Die Verbindung wird zentral in:

db/neo4j_connector.py

definiert.

Die standardmäßig verwendeten Zugangsdaten für die lokale Entwicklungsumgebung sind im Projekt fest hinterlegt:

URI:      bolt://localhost:7687
Benutzer: neo4j
Passwort: CfUsE_2025

Diese Werte müssen sowohl im Neo4j-Container als auch in einer lokal installierten Instanz übereinstimmen.

Verbindung prüfen:

macOS / Linux:

docker ps
docker logs neo4j

Windows (CMD):

docker ps
docker logs neo4j

Projektstart

Starten Sie das Hauptprogramm immer über die Datei app.py.

macOS / Linux:

cd /Pfad/zum/Projekt/CysSecMaTo_CFUSE
source .venv/bin/activate
python3 app.py

Windows (CMD):

cd C:\Pfad\zum\Projekt\CysSecMaTo_CFUSE
.\.venv\Scripts\activate
python app.py

Bemerkung

Der erste Start kann spürbar länger dauern, da: * die Qt-Oberfläche geladen wird, * die Datenbankverbindung aufgebaut wird, * Views wie Control-Embedding, Context-Retrieval, Mapping und Import initialisiert werden.

Projektstruktur (Kurzüberblick)

.
├── app.py                  # Einstiegspunkt
├── assets/                 # Styles & Markdown
├── config/                 # Prompt-/LLM-Konfigurationen
├── db/                     # Neo4j-Modelle, Queries, Import
├── logic/                  # Embeddings, Mapping, KI
├── retrieval/              # Fake-RAG & Tests
├── ui/                     # Qt-Frontend
├── tests/                  # Unittests
└── files/                  # Beispiel-OSCAL & JSON
