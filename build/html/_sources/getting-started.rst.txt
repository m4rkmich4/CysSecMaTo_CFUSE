Getting Started
===============

Einführung
-----------

C-FUSE ist ein lokal ausführbares Analyse- und Mapping-System für Cybersecurity-Standards.
Es kombiniert eine Neo4j-Graphdatenbank, KI-gestützte Ähnlichkeitsbewertungen und ein
modulares Qt-Frontend zur Visualisierung und Steuerung der Analyseprozesse.

Dieses Kapitel zeigt, wie das Projekt installiert, konfiguriert und gestartet wird.


Repository herunterladen
-------------------------

Das Projekt ist öffentlich verfügbar unter:

   https://github.com/m4rkmich4/CysSecMaTo_CFUSE.git

Klonen Sie das Repository in ein lokales Verzeichnis Ihrer Wahl:

.. code-block:: bash

   git clone https://github.com/m4rkmich4/CysSecMaTo_CFUSE.git
   cd CysSecMaTo_CFUSE


Systemvoraussetzungen
----------------------

* **Python:** Version 3.11 oder neuer
* **Abhängigkeiten:** siehe ``requirements.txt``
* **Neo4j:** Version 5.x (lokaler Docker-Container empfohlen)
* **Betriebssystem:** macOS, Linux oder Windows
* **Optional:** Lokales LLM über Ollama, wenn KI-Funktionen genutzt werden


Installation
------------

1. **Projektverzeichnis betreten**

.. code-block:: bash

   cd /Pfad/zum/Projekt/CysSecMaTo_CFUSE


2. **Virtuelle Umgebung erstellen**

macOS / Linux:

.. code-block:: bash

   cd /Pfad/zum/Projekt/CysSecMaTo_CFUSE
   python3 -m venv .venv
   source .venv/bin/activate

Windows (CMD):

.. code-block:: doscon

   cd C:\Pfad\zum\Projekt\CysSecMaTo_CFUSE
   python -m venv .venv
   .\.venv\Scripts\activate


3. **Abhängigkeiten installieren**

.. code-block:: bash

   pip install -r requirements.txt



4. Neo4j mit Docker Compose starten
-----------------------------------

Für C-FUSE wird Neo4j über eine vorbereitete Docker-Compose-Datei im Unterordner
``docker/`` gestartet. Diese setzt automatisch die passenden Zugangsdaten, Ports
und Persistenzoptionen für den Datenbankcontainer.

macOS / Linux:

.. code-block:: bash

   cd /Pfad/zum/Projekt/CysSecMaTo_CFUSE/docker
   docker compose up -d

Windows (CMD):

.. code-block:: doscon

   cd C:\Pfad\zum\Projekt\CysSecMaTo_CFUSE\docker
   docker compose up -d


Die von C-FUSE erwarteten Standard-Zugangsdaten sind:

.. code-block:: text

   URI:      bolt://localhost:7687
   Benutzer: neo4j
   Passwort: CfUsE_2025

Diese Werte sind sowohl in der ``docker-compose.yml`` als auch im Connector
``db/neo4j_connector.py`` definiert und müssen nur angepasst werden,
wenn von der Standardkonfiguration abgewichen wird.


Zugriff über den Browser (Neo4j Browser / Neo4j Workspace)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Nach dem Start mit Docker Compose kann die Neo4j-Oberfläche direkt im Browser geöffnet werden:

.. code-block:: text

   http://localhost:7474

Dort können Sie:

* den aktuellen Graphen ansehen,
* Abfragen testen,
* Knoten und Relationen inspizieren.

Melden Sie sich mit den voreingestellten Zugangsdaten an:

.. code-block:: text

   Benutzer: neo4j
   Passwort: CfUsE_2025



Projektstart
------------

Starten Sie das Hauptprogramm immer über die Datei ``app.py``.

macOS / Linux:

.. code-block:: bash

   cd /Pfad/zum/Projekt/CysSecMaTo_CFUSE
   source .venv/bin/activate
   python3 app.py

Windows (CMD):

.. code-block:: doscon

   cd C:\Pfad\zum\Projekt\CysSecMaTo_CFUSE
   .\.venv\Scripts\activate
   python app.py


.. note::

   Der erste Start kann **spürbar länger dauern**, da:
   * die Qt-Oberfläche geladen wird,
   * die Datenbankverbindung aufgebaut wird,
   * Views wie Control-Embedding, Context-Retrieval, Mapping und Import initialisiert werden.
