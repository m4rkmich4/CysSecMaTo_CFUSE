Getting Started
===============

Einführung
-----------

C-FUSE ist ein lokal ausführbares Analyse- und Mapping-System für Cybersecurity-Standards.
Es kombiniert eine Neo4j-Graphdatenbank, KI-gestützte Ähnlichkeitsbewertungen und ein
modulares Qt-Frontend zur Visualisierung und Steuerung der Analyseprozesse.

Dieses Kapitel zeigt, wie das Projekt installiert, konfiguriert und gestartet wird.


Repository herunterladen
-------------------------

Das Projekt ist öffentlich verfügbar unter:

   https://github.com/m4rkmich4/CysSecMaTo_CFUSE.git

Klonen Sie das Repository in ein lokales Verzeichnis Ihrer Wahl:

.. code-block:: bash

   git clone https://github.com/m4rkmich4/CysSecMaTo_CFUSE.git
   cd CysSecMaTo_CFUSE


Systemvoraussetzungen
----------------------

* Python 3.11 oder neuer
* Abhängigkeiten gemäß ``requirements.txt``
* Neo4j 5.x (über Docker Compose empfohlen)
* macOS, Linux oder Windows
* Optional: Lokales LLM über Ollama


Installation
------------

1. Projektverzeichnis betreten

macOS / Linux:

.. code-block:: bash

   cd /Pfad/zum/Projekt/CysSecMaTo_CFUSE

Windows (CMD):

.. code-block:: doscon

   cd C:\Pfad\zum\Projekt\CysSecMaTo_CFUSE


2. Virtuelle Umgebung erstellen und aktivieren

macOS / Linux:

.. code-block:: bash

   python3 -m venv .venv
   source .venv/bin/activate

Windows (CMD):

.. code-block:: doscon

   python -m venv .venv
   .\.venv\Scripts\activate


3. Abhängigkeiten installieren

.. code-block:: bash

   pip install -r requirements.txt

3.1 Für Windows ONLY sonst läuft sentence-transformers nicht (!)

.. code-block:: bash

   pip install tf-keras


4. Modelle lokal bereitstellen (Git + Git LFS)
----------------------------------------------

Damit C-FUSE vollständig lokal läuft, müssen die in ``/models`` referenzierten
Embedding-Modelle lokal verfügbar sein („all-MiniLM-L6-v2“ und „all-mpnet-base-v2“).

Git LFS ist hierfür erforderlich.

Git installieren (macOS / Linux / Windows)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

macOS Installer:
https://git-scm.com/download/mac

Linux (Debian/Ubuntu):

.. code-block:: bash

   sudo apt-get install git

Windows Installer:
https://git-scm.com/download/win


Git LFS installieren
~~~~~~~~~~~~~~~~~~~~

macOS / Linux:

.. code-block:: bash

   git lfs install

Windows (CMD):

.. code-block:: doscon

   git lfs install


Modelle herunterladen
~~~~~~~~~~~~~~~~~~~~~

Navigieren Sie in den ``models``-Ordner:

macOS / Linux:

.. code-block:: bash

   cd /Pfad/zum/Projekt/CysSecMaTo_CFUSE/models

Windows (CMD):

.. code-block:: doscon

   cd C:\Pfad\zum\Projekt\CysSecMaTo_CFUSE\models


Laden Sie die benötigten Modelle (bereits im Repository verlinkt):

.. code-block:: bash

   git clone https://huggingface.co/sentence-transformers/all-MiniLM-L6-v2
   git clone https://huggingface.co/sentence-transformers/all-mpnet-base-v2

Die Ordner müssen wie folgt aussehen:

.. code-block:: text

   models/
   ├── all-MiniLM-L6-v2/
   └── all-mpnet-base-v2/


Neo4j mit Docker Compose starten
---------------------------------

Die Neo4j-Instanz wird über die Datei ``docker/docker-compose.yml`` gestartet.

macOS / Linux:

.. code-block:: bash

   cd /Pfad/zum/Projekt/CysSecMaTo_CFUSE/docker
   docker compose up -d

Windows (CMD):

.. code-block:: doscon

   cd C:\Pfad\zum\Projekt\CysSecMaTo_CFUSE\docker
   docker compose up -d


Standard-Zugangsdaten:

.. code-block:: text

   URI:      bolt://localhost:7687
   Benutzer: neo4j
   Passwort: CfUsE_2025


Zugriff über den Browser (Neo4j Workspace)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Nach dem Start erreichbar unter:

.. code-block:: text

   http://localhost:7474

Melden Sie sich mit den oben genannten Zugangsdaten an.


Projektstart
------------

Starten Sie C-FUSE über ``app.py``.

macOS / Linux:

.. code-block:: bash

   cd /Pfad/zum/Projekt/CysSecMaTo_CFUSE
   source .venv/bin/activate
   python3 app.py

Windows (CMD):

.. code-block:: doscon

   cd C:\Pfad\zum\Projekt\CysSecMaTo_CFUSE
   .\.venv\Scripts\activate
   python app.py


.. note::

   Der erste Start kann etwas länger dauern, da die Qt-Oberfläche initialisiert
   und die Datenbankverbindung aufgebaut wird.


Projektstruktur (Kurzüberblick)
-------------------------------

.. code-block:: text

   .
   ├── app.py                  # Einstiegspunkt
   ├── assets/                 # Styles & Markdown
   ├── config/                 # Prompt- und KI-Konfigurationen

   ├── logic/                  # Embeddings, Mapping, KI-Module
   ├── retrieval/              # Fake-RAG-Module
   ├── ui/                     # Qt-Frontend
   ├── tests/                  # Unittests
   └── files/                  # Beispiel- und OSCAL-Kataloge
